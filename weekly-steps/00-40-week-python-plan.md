# 40-WEEK PYTHON LEARNING PROGRAM

## PHASE 1: FOUNDATIONS (Weeks 1-3)

### Week 1 - Basics and control flow
Variables, data types, print(), input().
if / elif / else, comparison and logical operators.
**Build:** a number guessing game.

### Week 2 - Loops and functions
for, while, break, continue, range().
def, parameters, return values, scope.
**Build:** FizzBuzz + temperature converter.

### Week 3 - Data structures, strings, file I/O
Lists, tuples, dicts, sets. Indexing, slicing.
String methods, f-strings, reading/writing files.
**Build:** a contact book that saves to file.

---

## PHASE 2: INTERMEDIATE (Weeks 4-12)

### Week 4 - Error handling
try / except / finally, common exceptions, raising errors.
**Build:** harden your week 3 contact book against bad input.

### Week 5 - Modules and packages
import, standard library (os, sys, math, random, datetime).
**Build:** a random password generator.

### Week 6 - OOP part 1
Classes, __init__, instance methods, attributes.
**Build:** a Bank Account class with deposit/withdraw.

### Week 7 - OOP part 2
Inheritance, super(), method overriding, dunder methods.
**Build:** extend Bank Account with SavingsAccount subclass.

### Week 8 - Comprehensions and functional tools
List/dict/set comprehensions, map(), filter(), lambda.
**Build:** a data cleaner for a list of messy strings.

### Week 9 - Working with CSV and JSON
csv module, json module, parsing and writing structured data.
**Build:** a script that reads a CSV and outputs a JSON summary.

### Week 10 - pip and virtual environments
pip, venv, requirements.txt. Install third-party libraries.
**Build:** use requests to fetch data from a public API.

### Week 11 - Testing
unittest or pytest basics. Writing test cases.
**Build:** write tests for your Bank Account classes.

### Week 12 - Debugging
pdb, breakpoints, IDE debugger, reading tracebacks.
**Build:** intentionally break a past project, debug it systematically.

---

## PHASE 3A: WEB TRACK (Weeks 13-18)

### Week 13 - Flask basics
Routes, templates (Jinja2), serving HTML.
**Build:** a simple multi-page website.

### Week 14 - Forms and databases
Handling form input, SQLite, SQLAlchemy basics.
**Build:** a guestbook that stores messages in a database.

### Week 15 - User authentication
Sessions, login/logout, password hashing.
**Build:** add user accounts to your guestbook.

### Week 16 - REST APIs
Returning JSON, HTTP methods, API design basics.
**Build:** a simple API for your guestbook data.

### Week 17-18 - Web project
Build a complete web app (e.g. task manager, blog, bookmarks).
Must include: database, auth, forms, and clean UI.

---

## PHASE 3B: DATA TRACK (Weeks 19-24)

### Week 19 - Pandas basics
DataFrames, Series, reading CSVs, basic operations.
**Build:** load and explore a dataset from Kaggle.

### Week 20 - Data cleaning
Handling nulls, duplicates, type conversion, renaming.
**Build:** clean a messy real-world dataset.

### Week 21 - Visualization
Matplotlib and Seaborn: line, bar, scatter, histograms.
**Build:** create 5+ charts from your cleaned dataset.

### Week 22 - Numpy and statistics
Arrays, vectorized operations, descriptive statistics.
**Build:** statistical summary report of a dataset.

### Week 23-24 - Data project
End-to-end analysis of a real dataset.
Must include: cleaning, exploration, visualization, insights.

---

## PHASE 3C: AUTOMATION TRACK (Weeks 25-30)

### Week 25 - File system automation
os, shutil, pathlib. Batch renaming, organizing, copying.
**Build:** a script that organizes downloads folder by file type.

### Week 26 - Web scraping
requests + BeautifulSoup, parsing HTML, handling pagination.
**Build:** scrape data from a public website into CSV.

### Week 27 - Browser automation
Selenium or Playwright. Interacting with dynamic pages.
**Build:** automate a repetitive web task you actually do.

### Week 28 - Scheduling and notifications
Cron/Task Scheduler, smtplib for email, logging.
**Build:** a script that runs daily and sends you a report.

### Week 29-30 - Automation project
Build a tool for a real task you do repeatedly.
Must include: error handling, logging, scheduling.

---

## PHASE 4: CAPSTONE AND MASTERY (Weeks 31-40)

### Week 31 - Code quality
PEP 8, linting (flake8/ruff), type hints, docstrings.
Refactor one past project to professional standards.

### Week 32 - Advanced testing
Mocking, fixtures, coverage, integration tests.
**Build:** achieve 80%+ test coverage on a past project.

### Week 33 - Project architecture
Project structure, config management, CLI with argparse/click.
**Build:** restructure a past project with proper architecture.

### Week 34-38 - Capstone project
Build one substantial project combining multiple tracks.
Examples:
- Web dashboard that scrapes data and visualizes it
- Automated data pipeline with web interface
- API that processes and analyzes uploaded files

Must include: clean code, tests, error handling, documentation.

### Week 39 - Polish and deployment
README, requirements.txt, Docker basics or cloud deployment.
Deploy your capstone so others can use it.

### Week 40 - Reflect and plan next steps
Review weak areas. Choose next direction:
algorithms, async Python, machine learning, new framework,
contributing to open source, or deeper into a track.
